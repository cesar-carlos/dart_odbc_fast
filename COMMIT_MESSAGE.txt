Initial commit: Enterprise ODBC Data Platform

Enterprise-grade ODBC data platform built with Rust (native engine) and Dart (clean, testable API).

## Architecture

- Clean Architecture with Domain/Application/Infrastructure layers
- Rust native engine for direct ODBC driver communication via FFI
- Dart FFI bindings with type-safe API
- No third-party ODBC packages - in-repo implementation

## Rust Engine (native/odbc_engine)

### Core Features
- Direct ODBC API access via odbc_api crate
- Binary protocol for efficient data transfer (reduces allocations)
- Streaming queries with chunked results
- Connection pooling via r2d2
- Transaction support with isolation levels and SQL savepoints
- Thread-safe error handling with connection-specific error isolation
- Structured error information (SQLSTATE, native codes, messages)

### Advanced Features
- Columnar data encoding/decoding
- Protocol compression
- Buffer arenas for memory management
- Disk spill for large result sets
- Metadata cache (TTL + LRU)
- Prepared statement cache (LRU)
- Driver-aware plugins (SQL Server, Oracle, PostgreSQL, MySQL, Sybase)
- Observability (metrics, logging, tracing)
- Security layer (secure buffers, secret manager, audit logger)
- Versioning (ABI, API, Protocol versions)

## Dart API (lib/)

### Domain Layer
- Entities: Connection, QueryResult, OdbcMetrics, PoolState, IsolationLevel
- Error hierarchy: OdbcError (ConnectionError, QueryError, ValidationError)
- Repository interfaces with Result<T> pattern

### Application Layer
- OdbcService: High-level API for all operations
- Service locator for dependency injection

### Infrastructure Layer
- Native ODBC connection wrapper
- FFI bindings with automatic library loading
- Binary protocol parser
- Bulk insert payload builder
- Parameter value serialization
- Streaming query implementation
- Catalog query wrapper
- Connection pool wrapper
- Prepared statement wrapper
- Transaction handle wrapper

### Core Utilities
- Structured logging
- Error conversion utilities
